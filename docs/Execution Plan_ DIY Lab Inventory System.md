# **Execution Plan: DIY Lab Inventory & Project Orchestrator**

This document outlines a modular approach to building the DIY Lab Inventory & Project Orchestrator. Each phase is designed to be testable and "vibe-coding" friendly.

## **Pre-requisites & Setup**

Before starting Phase 1, ensure your environment is ready:

* **Local Server:** Install XAMPP, WAMP, or MAMP to run PHP and MySQL locally.  
* **Database Tool:** Ensure you have access to phpMyAdmin (usually included with the local server).  
* **API Access:** Have your Gemini or OpenAI API key ready.  
* **File Structure:** Create a folder named diy\_lab in your web root (e.g., htdocs or www). Inside it, create an uploads/ folder.

## **Phase 1: The Foundation (Environment & Schema)**

**Goal:** Establish a robust database and secure connection.

* **Task 1.1: Database Setup:** Create a table inventory with the full schema: id, name, model, category, quantity, status, specs (TEXT/JSON), and image\_paths (TEXT).  
* **Task 1.2: Settings Table:** Create a table settings with setting\_key and setting\_value to store API keys and preferences securely via the UI.  
* **Task 1.3: Connectivity:** Create db.php for connection logic and index.php with a simple password gate.  
* **Test:** Can you access the page and see a "Connection Successful" message?

## **Phase 2: Comprehensive Dashboard & Data Entry**

**Goal:** Build the CRUD functionality with the full data schema.

* **Task 2.1: Full Entry Form:** Create add\_item.php with fields for *all* database columns (Name, Model, Category, Quantity, Status, and a Textarea for Specs).  
* **Task 2.2: Dashboard View:** Build dashboard.php to display items. Ensure the table is ready to show a thumbnail once images are added.  
* **Task 2.3: Delete Logic:** Add a delete function that is "image-aware" (it will check the image\_paths and delete physical files later).  
* **Test:** Add an item with full specs. Does it appear correctly on the dashboard?

## **Phase 3: Multi-Angle Image Handling**

**Goal:** Implement the gallery and file management system.

* **Task 3.1: Bulk Upload:** Update the form to allow multiple file uploads simultaneously.  
* **Task 3.2: JSON Path Storage:** Write logic to save images to /uploads and store their paths as a JSON array in the image\_paths column.  
* **Task 3.3: Item Detail Page:** Create item\_details.php to display the full image gallery and detailed specifications for a single component.  
* **Test:** Upload 3 photos of one part. Does the dashboard show the first one, and the details page show all three?

## **Phase 4: UI-Based AI Configuration**

**Goal:** Manage AI providers and keys without touching the code.

* **Task 4.1: Settings Interface:** Create settings.php where you can toggle between "Gemini" and "OpenAI" and save your API key to the settings table.  
* **Task 4.2: AI Proxy:** Build a helper function call\_ai\_api() that fetches the key from the database and handles the CURL request.  
* **Test:** Save your key in the UI. Run a test script to see if the system can retrieve that key and ping the AI successfully.

## **Phase 5: Automated Identification Workflow**

**Goal:** Use multiple images to identify hardware.

* **Task 5.1: Multi-Image AI Logic:** Update the "Auto-Identify" button to send *all* uploaded images of a component to the AI for better accuracy.  
* **Task 5.2: Parsing & Auto-Fill:** Use JavaScript to catch the AI's JSON response and automatically populate the Name, Model, and Specs fields in the form.  
* **Test:** Upload three angles of an ESP32. Does the AI accurately identify the model and fill the form for you?

## **Phase 6: Inventory Population & Physical Audit**

**Goal:** Transition from testing to a fully populated, real-world database.

* **Task 6.1: Physical Batching:** Group your physical items (e.g., all sensors, all microcontrollers, all passives).  
* **Task 6.2: Scanning Sprint:** Use the "Auto-Identify" tool to enter your actual stock.  
* **Task 6.3: Quality Control:** Manually review the "Specs" and "Model" generated by the AI for each real item.  
* **Task 6.4: Physical Mapping:** Add a "Location" field to the database and update items to match physical storage bins.  
* **Test:** Search for a specific part in your dashboard. Can you find it and its physical location?

## **Phase 7: Creative Engine & Acquisition Planning**

**Goal:** Generate projects and identify what you need to buy.

* **Task 7.1: Inventory Analysis:** Create projects.php to feed your entire inventory list into the AI.  
* **Task 7.2: Discovery & Links:** Use the prompt to generate projects that include Amazon/AliExpress links for any missing components.  
* **Task 7.3: Blueprint Generation:** Add a "Generate Blueprint" button that creates the full manual and code block for a specific project.  
* **Test:** Request a project. Does it suggest something you can build now? Do the "Missing Part" links work?

## **Phase 8: Interactive Assistant**

**Goal:** Add a conversational brainstorming layer.

* **Task 8.1: Chat Interface:** Create a sidebar for quick chatting.  
* **Task 8.2: Context-Aware Chat:** Ensure the chat sends your current inventory list as "context" so the AI knows what you have.  
* **Test:** Ask "What can I do with my extra LEDs?" Does it suggest a project based on your specific LED count?

## **Phase 9: Web Server Migration**

**Goal:** Move your system from your local machine to a live domain.

* **Task 9.1: Database Migration:** Export your .sql from localhost and import it into your live server via phpMyAdmin.  
* **Task 9.2: FTP Upload:** Upload all PHP files and create the uploads/ folder on the server.  
* **Task 9.3: Permissions:** Set uploads/ permissions to 755 or 777\.  
* **Task 9.4: SSL & Security:** Ensure the live site uses HTTPS to protect your API keys during transmission.  
* **Test:** Log in to your live URL. Does your inventory appear exactly as it did locally?
